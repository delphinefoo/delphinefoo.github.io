--- layout: post title: Saving a dateTime object in a MySQL database categories: - weirdJS - MySQL - dateTime ---

I'm working with a MySQL database in my current project, and I have a "date" column in the table. I'm using a datepicker from Material-UI for React, which provides the date as a Date object. However, when I went to save the Date object into our MySQL database, I kept getting an error saying "Incorrect dateTime value '2016-02-07T17:00:00Z00:00'".

I read up on MySQL's dateTime type, and tried saving the date provided by the datepicker as a string, as a new Date object, as stringified JSON... No dice. What worked for me? Importing moment.js, and formatting the date object provided by the datepicker with the function moment('my date object here').format('YYYY-MM-DD HH:mm:ss'). The MySQL database liked that format!
